# دليل الصلاحيات والتحديثات الجديدة

## خطوات التطبيق

### 1. تنفيذ سكريبت SQL
افتح phpMyAdmin أو MySQL وقم بتنفيذ ملف `update_roles.sql`

```sql
-- سيقوم السكريبت بـ:
-- 1. إضافة أعمدة last_action_by و last_action_type لجدول complaints
-- 2. إنشاء جدول status_change_requests لطلبات تغيير الحالة
-- 3. تعديل صلاحيات المستخدمين لتشمل: super_admin, admin, supervisor, agent
-- 4. تحويل أول مستخدم إلى super_admin تلقائياً
```

## الصلاحيات الجديدة

### 1. Super Admin (مدير عام)
- الصلاحية الكاملة على النظام
- **فقط** يستطيع إضافة وحذف المستخدمين
- يرى كل الشكاوى والطلبات
- يغير الحالات مباشرة بدون موافقة
- يوافق على طلبات تغيير الحالة من الموظفين

### 2. Admin (مدير)
- يرى كل الشكاوى والطلبات
- يغير الحالات مباشرة
- يوافق على طلبات تغيير الحالة
- **لا** يستطيع إضافة أو حذف المستخدمين

### 3. Supervisor (مشرف)
- يرى كل الشكاوى والطلبات
- يتابع الموظفين
- يوافق على طلبات تغيير الحالة
- **لا** يستطيع إضافة أو حذف المستخدمين
- **لا** يستطيع تغيير الحالات مباشرة

### 4. Agent (موظف)
- يرى **فقط** الشكاوى المُكلف بها
- يضيف ردود وملاحظات داخلية
- **يطلب** تغيير الحالة (يحتاج موافقة من Admin/Supervisor)
- **لا** يرى قائمة المستخدمين أو التقارير

## الميزات الجديدة

### 1. طلبات تغيير الحالة
- الموظف يضغط على زر "طلب تغيير الحالة"
- يختار الحالة الجديدة ويكتب سبب التغيير
- تظهر في صفحة "طلبات التغيير" للأدمن والمشرف
- الأدمن/المشرف يوافق أو يرفض

### 2. تسجيل آخر معاملة
- كل تغيير يتم تسجيل من قام به
- يظهر في تفاصيل الشكوى: "آخر تحديث بواسطة: [اسم المستخدم]"
- أنواع المعاملات:
  - تغيير الحالة
  - إضافة رد
  - إضافة ملاحظة داخلية
  - موافقة على تغيير الحالة

### 3. فلترة الشكاوى حسب الصلاحية
- الموظف يشوف بس الشكاوى المكلف بها
- الأدمن والمشرف يشوفوا كل حاجة

### 4. تصدير أرقام التليفونات
- زر جديد في صفحة الشكاوى
- يصدر ملف CSV فيه: الاسم، رقم التليفون، الكود، العنوان، التاريخ

## واجهة المستخدم

### للموظف (Agent)
- قائمة الشكاوى المكلف بها فقط
- في صفحة التفاصيل:
  - زر "طلب تغيير الحالة" بدلاً من تغيير مباشر
  - يظهر نموذج: اختر الحالة + اكتب السبب

### للأدمن والمشرف
- قائمة "طلبات التغيير" في القائمة الجانبية
- يظهر عدد الطلبات المعلقة
- زر "موافقة" و "رفض" لكل طلب

### للـ Super Admin
- قائمة "المستخدمين" تظهر فقط له
- يستطيع إضافة وحذف المستخدمين

## ملاحظات مهمة

1. المستخدم الأول في قاعدة البيانات يتم تحويله تلقائياً إلى Super Admin
2. لا يمكن لأي أدمن حذف نفسه
3. الموظف يجب أن يكون مُكلف بالشكوى ليستطيع طلب تغيير حالتها
4. كل طلب تغيير يحتاج سبب (اختياري ولكن مفضل)
